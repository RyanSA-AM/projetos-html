Entendendo Comunicação Client x Server

História da Web

Neste módulo do curso o tema tratado foi a história da Web, desde seu surgimento até a sua evolução para chegar no nível em que ela está hoje. Foram abordados tópicos como os primeiros computadores funcionais inventados na década de 40, o surgimento da internet e os principais nomes responsáveis pelo surgimento da Web naquela época.

O que são Clients

No segundo módulo do curso, foi abordado o tema Cliente no linguajar Web. No contexto da linguagem web, o termo "client" (cliente, em português) refere-se a um componente fundamental na arquitetura cliente-servidor. O cliente é um software ou dispositivo que solicita recursos ou serviços a um servidor por meio da Internet. Em outras palavras, é o programa ou aplicativo usado por um usuário para interagir com um servidor remoto e acessar conteúdo, serviços ou dados disponibilizados pela web.

O que são Servers

O terceiro módulo, abordou o tema Servidores. Servidores desempenham um papel fundamental na infraestrutura da internet e na entrega de serviços e recursos online. Um servidor é um computador ou sistema especializado que recebe solicitações de outros dispositivos, como clientes, e fornece respostas apropriadas. Ele é responsável por armazenar, processar e fornecer dados, aplicativos, sites e outros conteúdos para os clientes que os solicitam.

Linguagens de Programação

Por fim, o último tópico estudado foi a respeito das linguagens de programação, explicando o conceito de linguagem de programação e client side x server side. ResumidamenteServer-side e client-side referem-se a duas áreas distintas de processamento em uma aplicação web. No lado do servidor (server-side), o processamento é realizado no servidor antes de enviar a resposta ao cliente. Isso envolve o uso de linguagens de programação como PHP, Python ou Java, que são executadas no servidor para processar solicitações, acessar bancos de dados, gerar conteúdo dinâmico e enviar respostas ao cliente. Por outro lado, no lado do cliente (client-side), o processamento ocorre no navegador do cliente, utilizando tecnologias como HTML, CSS e JavaScript. O cliente faz solicitações ao servidor, recebe conteúdo estático e scripts JavaScript, que são executados no navegador para manipular a interface do usuário, validar formulários, enviar requisições adicionais ao servidor e exibir conteúdo dinâmico. Essa divisão entre server-side e client-side permite uma distribuição de tarefas eficiente, proporcionando uma experiência interativa e responsiva para o usuário enquanto alivia a carga de processamento do servidor.
